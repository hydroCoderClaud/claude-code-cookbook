# 工作报告功能使用说明

## 功能概述

工作报告功能允许用户上传静态 HTML 文件作为工作日志，系统会以美观的方式展示这些报告。

## 使用步骤

### 1. 准备 HTML 文件

创建一个完整的 HTML 文件，包含所有样式和内容。参考 `example-report.html` 作为模板。

**注意事项：**
- 文件大小不超过 50MB
- 必须是 `.html` 或 `.htm` 格式
- 建议使用内联 CSS 样式（`<style>` 标签）
- 避免使用外部资源链接（如外部 CSS、JS、图片），除非确保可访问

### 2. 上传报告

1. 登录系统
2. 点击顶部导航栏的"上传工作报告"
3. 填写报告标题和描述
4. 拖拽或点击上传 HTML 文件
5. 添加相关标签（可选）
6. 点击"提交"

### 3. 查看报告

- 在首页可以看到所有工作报告（标签为"工作报告"）
- 点击报告标题进入详情页
- 报告内容通过 iframe 展示，保持原有样式
- 支持评论和分享功能

### 4. 管理报告

- 作者和管理员可以编辑报告的标题、描述和标签
- 作者和管理员可以删除报告
- HTML 文件本身不可修改，如需更换请删除后重新上传

## 筛选和搜索

在首页可以：
- 通过类型筛选器选择"工作报告"
- 使用标签筛选特定主题的报告
- 通过搜索框搜索报告标题和描述
- 按时间范围筛选（近一日、近一周、近一月、近一年）

## 示例模板

项目根目录下的 `example-report.html` 提供了一个美观的工作日志模板，包含：
- 渐变色头部
- 数据概览卡片
- 任务列表（已完成、进行中、待开始）
- 状态标签
- 响应式布局

你可以基于这个模板创建自己的工作报告。

## 技术实现

- 后端：使用 multer 处理文件上传，存储在 `backend/data/reports/` 目录
- 前端：使用 iframe 展示 HTML 内容，自动调整高度
- 数据库：items 表新增 `html_file` 字段存储文件名
- 安全：仅允许上传 HTML 文件，禁止可执行文件

## 权限说明

- **游客**：可以浏览和查看所有工作报告
- **普通用户**：可以上传、删除自己的报告
- **管理员**：可以删除任何报告
